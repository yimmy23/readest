**FR #5062: sync to multiple providers at once** (Readest Cloud + Google Drive + WebDAV + S3 + OneDrive, any subset). PR **#5122 MERGED 2026-07-15** (30 commits); worktree + local branch removed. Reverses the exclusive model from [[cloud-sync-provider-selection-plan]] (#4975/#4976) — but keeps its #4380/#4959 fixes intact (dual-run is now opt-in).

**Core design:** NO migration. `readestCloud.enabled` is a real flag but DELIBERATELY absent from `DEFAULT_SYSTEM_SETTINGS` (loadSettings shallow-spreads defaults, so a default would flip native sync back on for existing WebDAV/Drive users). `isReadestCloudEnabled(s) = s.readestCloud?.enabled ?? !hasAnyThirdPartyEnabled(s)` — the `??` reproduces the old exclusive behaviour for an absent flag. Fresh user → `getCloudSyncProviders` = `['readest']`. `getCloudSyncProvider` (singular) DELETED; new API is `getCloudSyncProviders`/`getActiveFileSyncBackends`/`resolveCloudSyncGate`→`{readest,backends,paused}`. `getEnabledFileSyncBackends` MOVED providerRegistry→cloudSyncProvider.ts.

**Execution:** `runFileLibrarySyncPass` (renamed from runActiveFileLibrarySync) runs each enabled backend sequentially under a mutex HELD across the whole pass via `fileSyncStore.switchSync` (no release between backends, else auto-sync double-reconciles). Per-backend failure isolated (AUTH_FAILED records lastError, pass continues). `endSync` releases the LIVE activeKind (leak-proof). providerRegistry cache = Map per kind (was single slot). `canBackendRun(kind)` skips expired web-Drive token. Reader `useFileSync` drives every backend; PULL CHAINS `working = mergedConfig` (else a mirror's data drops); sub-toggle mask applied PER BACKEND inside the loop (my plan's wantProgressAny/wantNotesAny OR was a real leak bug — fixed in review). `authNotifiedRef` is a Set per kind. lastSyncedAt batched to 1 write.

**Gates flipped:** native book/progress/note gate on `!isReadestCloudEnabled` (was "any third-party enabled"). `isReadestCloudStorageActive` kept name/sig, new meaning = "readest enabled" → ~8 call sites (transferManager/ingest/BookItem/etc.) unchanged. Fleet probe runs only when Readest Cloud OFF, anchors on `readestCloud.disabledAt` (min providerSelectedAt fallback). Broadcast (`settingsSync`) + backup blacklist carry `readestCloud`; guard `if(settings.readestCloud)` so undefined never coerces to false.

**UI:** checkboxes not radios (`role=group`); `canToggleCloudProvider` helper — a lapsed-plan user can ALWAYS switch a provider OFF (else trapped); `checked={!!user && readestEnabled}`; zero-provider empty state; `getThirdPartyRowStatus` gains `booksBackedUpElsewhere` so "Book file uploads off" only warns when NOTHING covers files. Per-provider connect/disconnect via `persistCloudProviderEnabled(kind,enabled)` (old disconnect nuked ALL four slices — the bug this prevents). `useBookTransferActions` hook: upload reaches EVERY destination, download prefers Readest Cloud when `book.uploadedAt`.

**Live-verified in Chrome (web build)** 2026-07-15: 3 providers (Readest Cloud + WebDAV + Google Drive) active at once, enabling one doesn't disable others, checkboxes not radios, persists across reload, no console errors, zh-CN renders. Two UX refinements added after: hide "Premium" badge from entitled users (`premiumBadge` gated on `isCloudSyncPremium`, suppressed while plan loads to avoid flashing premium users); SettingsMenu sync row shows COUNT not names when >1 provider (`Library sync via {{count}} providers` — `{{count}}` triggers i18next plurals: en manual `_one`/`_other`, scanner scaffolds CLDR categories per locale, filled x33; common case is exactly 2 where Slavic/Arabic paucal/dual differ).

**Open follow-ups:** live verify NATIVE desktop + both-remotes-in-one-pass via network trace + expired-Drive-doesn't-stop-WebDAV; reader PUSH side still over-shares whole config per backend regardless of that backend's syncNotes/syncProgress (pre-existing, now x mirrors — pull side fixed); Readest-only + autoUpload:false = books nowhere, no row warns (pre-existing). **Low-confidence i18n for native pass:** `bo` (Tibetan) all strings; `ar` + `sl` count-plural forms (dual/paucal); Slavic paucal forms (ru/uk/pl `Library sync via {{count}} providers`).
